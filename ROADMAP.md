# Roadmap — новые фичи Егерь ИИ

Дата планирования: 2026-05-17

---

## F1 — Фото-определитель рыбы через ИИ (Claude Vision)
**Сложность:** Средняя | **Приоритет:** 🔥

### Что меняется:
- `functions/index.js` — новая функция `identifyFish` (Claude Vision API, `claude-sonnet-4-6`)
- `src/components/CatchModal.jsx` — кнопка "Определить ИИ" после выбора фото, заменяет пиксельный анализ
- `src/screens/DiaryScreen.jsx` — кнопка определения на существующих уловах без вида рыбы

### Детали реализации:
- Изображение → base64 → Cloud Function → Claude Vision (claude-sonnet-4-6 с vision)
- Промпт: вид рыбы, примерный вес по размеру, советы
- Firestore: результат кэшируется в документ улова

## F2 — Оффлайн-режим для дневника
**Сложность:** Высокая | **Приоритет:** 🔥

### Что меняется:
- `src/lib/offlineQueue.js` — новый модуль: IndexedDB очередь операций
- `src/screens/DiaryScreen.jsx` — перехват save/delete → пишем в IndexedDB если офлайн
- `src/App.jsx` — при `online` событии: дренируем очередь в Firestore
- `src/components/CatchModal.jsx` — сохранение офлайн с уведомлением

### Детали реализации:
- `indexedDB` → БД `eger-offline`, стор `catches-queue`
- Операции: `{type: 'add'|'update'|'delete', payload, ts}`
- При синхронизации: применяем по порядку timestamp
- Счётчик `offlinePending` уже есть в App.jsx

## F3 — Push-уведомления при изменении клёва
**Сложность:** Средняя | **Приоритет:** 🔥

### Что меняется:
- `functions/index.js` — Cloud Scheduler: каждый час проверяем давление, при изменении >3 мм отправляем FCM
- `src/screens/ProfileScreen.jsx` — UI настройки уведомлений (клёв, уловы рядом, новости)
- `firestore.rules` — коллекция `notification_prefs`

### Детали реализации:
- Scheduler: `fetchWeather → сравнить с предыдущим → если дельта > порога → FCM multicast`
- Типы: `bite_change` (давление), `nearby_catch` (улов в 20 км), `news` (новый отчёт)
- Настройки пользователя в Firestore `notification_prefs/{uid}`
