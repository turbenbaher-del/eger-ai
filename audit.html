UX/UI Аудит

1.1 Навигация и информационная архитектура

ХОРОШО

Нижняя таб-навигация

5 вкладок (Главная, ИИ-чат, Новости, Сообщество, Профиль) — оптимальное количество. Активная вкладка выделена цветом и подписью. Иконки самодокументируемые.

ХОРОШО

Сплэш-экран

Анимированный логотип + спиннер + плавное исчезновение через 600 мс. Создаёт ощущение быстрого старта. Не блокирует UI после загрузки.

УЛУЧШИТЬ

Отсутствие breadcrumbs в DM

При открытии личных сообщений нет явного индикатора «где я» в иерархии. Кнопка «←» есть, но визуальный контекст слабый.

УЛУЧШИТЬ

Глубокая вложенность в профиле

Раздел «Дневник уловов» внутри профиля — пользователь может не найти его быстро. Рекомендуется добавить вкладку или плавающую кнопку добавления улова.

1.2 Первый запуск (Onboarding)

КРИТИЧНО

Отсутствует онбординг

Новый пользователь попадает сразу на экран с погодой. Нет обучающего overlay, подсказок «что делать дальше» или приветственного сообщения. Высокий риск оттока в первую сессию.

УЛУЧШИТЬ

Нет мотивации зарегистрироваться

Незарегистрированный пользователь видит погоду, но при попытке написать в чат получает запрос на вход. Отсутствует объяснение ценности регистрации.

1.3 ИИ-чат (ChatScreen)

ОТЛИЧНО

Telegram-like функции

Reply-to, свайп для ответа, лонг-пресс меню, реакции (6 эмодзи), группировка сообщений, разделители дат, тики прочтения, scroll FAB — полноценный мессенджер-UX.

ХОРОШО

Быстрые кнопки запросов

8 тематических кнопок под полем ввода (прогноз клёва, точки, луна и др.) снижают когнитивную нагрузку. Особенно полезно при первом использовании.

ХОРОШО

Мультимодальный ввод

Текст, голосовые сообщения (с таймером), фото из галереи, видео-кружки, прикрепление файлов — покрывает все сценарии мобильного общения.

УЛУЧШИТЬ

Лимит AI-запросов не виден

Пользователь не знает сколько бесплатных запросов к Claude API осталось. Нет индикатора. При исчерпании лимита ответ будет молчаливо деградирован до локального бота.

1.4 Чат сообщества (CommunityScreen)

ХОРОШО

Поиск по сообщениям

Живой поиск с подсветкой совпадений в тексте, именах и никнеймах. Счётчик результатов. Анимированное появление строки поиска.

ХОРОШО

Прямые сообщения (DM)

Нажатие на имя/аватар → открывает DM-overlay. Чат сохраняется в Firestore, история персистентна. Реальное время через onSnapshot.

ХОРОШО

Медиа-контент

Фото с увеличением, видео-кружки, голосовые сообщения с прогресс-баром. Загрузка через Firebase Storage.

УЛУЧШИТЬ

Нет пагинации в DM

DM-чат загружает последние 50 сообщений без lazy-loading. При длинной переписке — проблемы с производительностью.

1.5 Главный экран (HomeScreen)

ОТЛИЧНО

Прогноз клёва BiteArc

Уникальный SVG-виджет с дуговым индикатором 1–10. Цвет зависит от балла (зелёный/жёлтый/красный).

ХОРОШО

Почасовой Sparkline

Интерактивный SVG-график давления/температуры с маркером текущего часа и градиентной заливкой.

ХОРОШО

Лунная фаза + геолокация

Фаза луны с агрономическим советом. Геолокация для ближайших точек рыбалки и сортировки по расстоянию.

Иконки

Все иконки — кастомные inline SVG-компоненты (Lucide-style). Stroke-based, масштабируемые, не зависят от сторонних шрифтов. Единый стиль: strokeWidth=2, strokeLinecap=round.

Дизайн-паттерны

СИЛЬНАЯ СТОРОНА

Glass-morphism

Единый glass() helper: backdrop-filter:blur(24px), rgba surface, border 1px. Корректно деградирует на старых iOS.

СИЛЬНАЯ СТОРОНА

Micro-анимации

fadeUp, slideUp, newsSlideIn, fabPulse, pulseGlow — все с корректными duration (200–600мс). Не раздражают и не тормозят.

СИЛЬНАЯ СТОРОНА

Токены дизайна

Объект C{} как единый источник истины для цветов. Все компоненты используют C.accent, C.muted и т.д. — легко менять тему.

УЛУЧШИТЬ

Tap targets на мобайл

Часть мелких кнопок (теги реакций, временные метки) имеют tap-area менее 44×44px — минимум Apple HIG. Риск для пальцев крупного размера.

УЛУЧШИТЬ

Плотность информации

HomeScreen перегружен: погода + BiteArc + давление + луна + Sparkline + магазины на одном экране. Рекомендуется карусель или аккордеон.

ХОРОШО

Единообразие бордеров

border-radius 12–20px консистентно. Внутренние элементы мельче (8–12px), карточки — 16–20px. Визуальная иерархия сохранена.

Найденные проблемы и рекомендации

🔴

ВЫСОКИЙ: Нет React ErrorBoundary

Любая необработанная ошибка в компоненте ломает всё приложение. Пользователь видит белый экран без возможности восстановления.

Решение: обернуть App в ErrorBoundary с кнопкой «Перезагрузить». 10 строк кода.
🔴

ВЫСОКИЙ: Babel standalone в production

type="text/babel" означает что JSX компилируется в браузере каждый раз. Добавляет 800–1200 мс к первому запуску. Babel.min.js весит ~800 КБ.

Решение: предкомпилировать JSX через Vite/CRA. Переход потребует рефактора, но ускорит старт в 3–5 раз.
🔴

ВЫСОКИЙ: Нет онбординга

Новый пользователь не понимает что делать. Конверсия в первое действие будет низкой. Нет ни одной подсказки или туториала.

Решение: 3-шаговый overlay при первом запуске (localStorage firstVisit). Ключевые фичи с анимацией.
🟡

СРЕДНИЙ: Яндекс.Карты без проверки загрузки

Если SDK не загрузился (медленный интернет / блокировщик) — MapScreen крашится с ReferenceError. Нет guard-а.

Решение: typeof ymaps !== 'undefined' перед инициализацией + fallback UI «Карта недоступна».
🟡

СРЕДНИЙ: Нет индикатора лимита AI

Пользователь не видит сколько запросов к Claude API осталось. При исчерпании молчаливый fallback на локального бота без объяснения.

Решение: читать /ai_usage/{userId} в ChatScreen, показывать прогресс-бар «X/10 запросов».
🟡

СРЕДНИЙ: Нет подтверждения удаления в ChatScreen

В AI-чате удаление сообщения через контекст-меню происходит без confirm(). В Community есть confirm для удаления отчётов, но не для сообщений.

Решение: добавить window.confirm() или Undo-toast на 4 секунды перед реальным удалением.
🟡

СРЕДНИЙ: DM — нет lazy loading старых сообщений

DMScreen загружает последние 50 сообщений одним запросом. При длинной переписке — медленно и дорого.

Решение: cursor-based пагинация через Firestore startAfter(). Загрузка «ещё 20» при скролле вверх.
🔵

НИЗКИЙ: Tap targets <44px

Кнопки реакций, временные метки, кнопка «×» закрытия — меньше рекомендуемых 44×44 px Apple/Google.

Решение: добавить padding или min-width/min-height: 44px на интерактивных элементах.
🔵

НИЗКИЙ: Weather error без уведомления

При сбое Open-Meteo fetch — молчание. Пользователь думает что данные свежие, хотя они могут быть устаревшими.

Решение: хранить timestamp последнего успешного запроса, показывать «Данные от ЧЧ:ММ» с предупреждением если >30 мин.
🔵

НИЗКИЙ: Нет индикатора загрузки чата Community

При первом открытии Community — пустой экран на время загрузки сообщений. Нет скелетона или спиннера.

Решение: skeleton-placeholder из 3–5 шиммер-строк пока loading===true.
